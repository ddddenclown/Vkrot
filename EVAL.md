# Проверка качества

Я прогнал API на треках из `unbake-vocals.zip` и отдельно сравнил результат с текстами песен.

Важно: `confidence` из WhisperX не считаю настоящей точностью. Это просто внутренняя уверенность модели. Реальная проверка - это сравнение с reference lyrics.

Что лежит в репозитории:
- сырые ответы API: `results/*.json`
- сводка по прогонам: `results/summary.json`
- автоматическая оценка: `results/eval_summary.json`
- скрипт для пересчета: `scripts/evaluate_results.py`

Прогон был на:
- `WHISPER_MODEL=small`
- `WHISPER_BATCH_SIZE=8`
- локальный API `http://127.0.0.1:8001`

Эталонные тексты я не кладу в репозиторий. Подтягивал их из LRCLIB по id.

## Что проверял

`WER` - сколько ошибок по словам. Чем ниже, тем лучше.

`CER` - сколько ошибок по символам. Полезно для языков, где модель может угадать слово частично.

`insertions / hyp word` - грубый сигнал галлюцинаций. Если число большое, модель добавляет лишние слова, которых нет в эталоне.

`mean confidence` - средняя уверенность WhisperX по словам. Это не accuracy, а только дополнительный сигнал.

`line median error` - примерная ошибка тайминга по строкам, если в LRCLIB был synced lyrics.

`line precision@3s` - какая доля строк попала в окно до 3 секунд.

Это не идеальная word-level разметка. Но для быстрого sanity-check уже видно, где модель работает нормально, а где разваливается.

## Что видно

Whisper small работает нестабильно(и это очевидно, тк модель small).

На части треков результат нормальный для прототипа. Например, `SOLICITADO` дал `WER=0.2279`, `BRUCE WAYNE` - `WER=0.3457`. Это не идеально, но уже похоже на рабочий fallback.

Есть треки, где текст узнаваемый, но ошибок много. `rockstar`, `Place de la Republique`, `Last of Us`, `PHARAOH` попадают примерно в эту зону.

Есть явный провал: `BELLAKEO` с `WER=0.9242`. Там модель сильно поплыла, confidence тоже низкий. Это хороший пример того, почему нельзя просто взять Whisper и считать задачу закрытой (как писалось ранее).

С таймингами тоже не все ровно. У `PHARAOH` тайминг по строкам получился близкий, а у `Bi-2` ошибка большая: median `6.315s`, `precision@3s = 0.1429`.

## Вывод

WhisperX можно оставить как fallback для каверов, демок и неизвестных треков.

Но для известных песен лучше не распознавать текст с нуля. Надежнее сначала понять, что это за трек, взять known lyrics и сделать forced alignment. Это должно уменьшить галлюцинации и улучшить sync.

Именно поэтому основной дизайн в `README_BIG.md` не строится вокруг одного ASR-пайплайна. По этим цифрам видно, что простой `Whisper small -> align` иногда работает, а иногда дает слишком грязный результат.

ВСе результаты лежат в `results/eval_summary.json`
